Chapter 7: Semantic HTML Essentials

Why Semantic HTML Matters

Key Benefits:
  • Improved accessibility
  • Better SEO rankings
  • Easier code maintenance
  • Enhanced device compatibility

Core Semantic Tags

<header>Site header</header>
<nav>Main navigation</nav>
<main>Primary content</main>
<article>Self-contained content</article>
<section>Thematic grouping</section>
<aside>Related content</aside>
<footer>Page footer</footer>

Semantic Document Structure

<body>
    <header>
        <h1>Website Title</h1>
        <nav>...</nav>
    </header>
    
    <main>
        <article>
            <h2>Blog Post Title</h2>
            <section>
                <h3>Chapter 1</h3>
                <p>Content...</p>
            </section>
        </article>
        
        <aside>
            <h2>Related Posts</h2>
            <ul>...</ul>
        </aside>
    </main>
    
    <footer>
        <p>Copyright information</p>
    </footer>
</body>

Accessibility Landmarks

<div role="banner">Header content</div>
<div role="navigation">Nav links</div>
<div role="main">Primary content</div>
<div role="complementary">Sidebar</div>
<div role="contentinfo">Footer content</div>

Semantic Text Elements

<time datetime="2023-09-01">September 1st</time>
<address>Contact information</address>
<blockquote cite="source">Quote text</blockquote>
<figure>
    <img src="chart.jpg" alt="Sales chart">
    <figcaption>Monthly sales data</figcaption>
</figure>

Microdata & SEO

<div itemscope itemtype="https://schema.org/Person">
    <span itemprop="name">John Doe</span>
    <span itemprop="jobTitle">Web Developer</span>
    <a itemprop="email" href="mailto:john@example.com">Email</a>
</div>
Schema.org Types:
  • Article
  • LocalBusiness
  • Product
  • Event
  • Recipe

Common Semantic Mistakes

Avoid:
  • Using div for everything
  • Multiple main elements
  • Empty heading tags
  • Incorrect nesting (e.g., header inside section)
