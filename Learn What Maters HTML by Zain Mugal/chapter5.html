Chapter 5: HTML Forms Deep Dive

Form Basics

<form action="/submit" method="POST">
    <!-- Form elements -->
</form>
Key Attributes:
  • action: Submission endpoint
  • method: GET (default) or POST
  • enctype: For file uploads (multipart/form-data)

Essential Form Elements

Element Purpose Example
<input> Generic input field <input type="text" name="username">
<textarea> Multi-line text <textarea name="bio"></textarea>
<select> Dropdown list <select name="country">...</select>
<button> Form submission <button type="submit">Send</button>

Input Types Deep Dive

<!-- Common input types -->
<input type="email" required>
<input type="password" minlength="8">
<input type="date">
<input type="file" accept=".pdf">
<input type="range" min="0" max="100">
Modern Input Types:
  • color: Color picker
  • search: Search field
  • tel: Telephone number
  • url: URL validation
  • datetime-local: Date/time picker

Form Validation

<input type="email" required 
       pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,4}$">

Advanced Form Features

Fieldset & Legend

<fieldset>
    <legend>Shipping Address</legend>
    <!-- Form fields -->
</fieldset>

Data List & Auto-complete

<input list="browsers">
<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
</datalist>

Accessibility Essentials

Must-Have Features:
  • Always use <label> elements
  • Add aria-describedby for complex fields
  • Use fieldset for grouped controls
  • Implement proper tab order
